# 用事务重试兜住串行化冲突

高并发写数据库时，串行化失败、死锁回滚等错误并不少见。若这类错误具备可重试特征，可以在事务边界统一重试，而不是把重试逻辑散落到业务代码里。

## 设计要点

- `Runner` 统一负责 `Begin`、`Commit` 和 `Rollback`
- `IsRetryable` 由调用方注入，便于适配不同数据库错误码
- 既处理业务执行阶段的可重试错误，也处理提交阶段的冲突错误

## 示例说明

接数据库驱动时，可以把 PostgreSQL 的串行化失败、MySQL 的死锁错误映射到 `IsRetryable`。如果事务内部包含外部副作用，必须保证这些副作用本身也具备幂等性。

## 运行方式

```bash
go test ./daily/2026/06/30/tx_retry
```
