# 用超时预算拆分下游调用时间

一个接口常常要依次访问缓存、数据库和 RPC。若只给总请求设置超时，某一步耗时失控时会把后续步骤全部挤压掉。这个示例把剩余时间按权重和最小保障时间拆给每个步骤。

## 设计要点

- 先读取 `context` 的总 deadline，再计算剩余预算
- 用 `MinShare` 保证关键步骤至少拿到基本时间
- 用 `Weight` 分配剩余预算，体现不同下游的重要性

## 示例说明

实际业务里可以把分配结果继续传给 `context.WithTimeout`，为缓存、数据库和 RPC 构造各自子上下文，避免单个慢步骤吞掉整条链路的超时空间。

## 运行方式

```bash
go test ./daily/2026/06/30/timeout_budget
```
