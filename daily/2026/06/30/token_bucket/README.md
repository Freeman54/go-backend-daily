# 用令牌桶限制突发流量

令牌桶适合接口限流、消息消费配额和后台任务节流。它允许短时间突发，但会按照固定速率补充令牌，避免下游被持续高并发压垮。

## 设计要点

- `capacity` 决定最多允许多少突发请求
- `refillEvery` 决定令牌恢复速度
- `AllowN` 支持一次消费多个令牌，便于表达批量请求

## 示例说明

真实服务里通常会把 `Bucket` 挂到用户、租户或接口维度。若要分布式共享额度，可以把当前逻辑迁移到 Redis 或网关层。
