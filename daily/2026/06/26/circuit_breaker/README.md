# 用断路器隔离不稳定下游

当数据库、缓存或外部接口已经持续失败时，继续无脑重试只会放大故障。这个示例实现了一个最小断路器，用来在失败达到阈值后快速失败，并在冷却窗口结束后放行一次探测请求。

## 状态流转

- `closed`：正常放流量，统计失败次数
- `open`：达到阈值后直接拒绝请求
- `half-open`：冷却时间结束后允许一次探测，成功则关闭，失败则重新打开

## 示例说明

`Breaker.Execute` 封装了状态判断和故障计数逻辑，调用方只需要把真正的下游访问包装成一个函数交给它执行。

## 运行方式

```bash
go test ./daily/2026/06/26/circuit_breaker
```
