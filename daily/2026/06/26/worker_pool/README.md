# 用有界 worker pool 控制并发任务

这个示例演示后端常见的有限并发模型：固定数量 worker 从任务队列中拉取任务，既避免一次性打爆下游，也能通过 `context.Context` 在错误或取消时尽快停机。

## 适用场景

- 批量调用外部接口，需要限制并发数
- 一次性处理大量任务，但又要保持结果与输入顺序一致
- 上游请求取消后，希望后台任务停止继续调度

## 示例说明

`Pool.Run` 做了三件事：

1. 固定数量 worker 并行执行任务
2. 结果按输入索引回填，保证输出顺序稳定
3. 任务报错或 context 取消时，停止继续投喂新任务

## 运行方式

```bash
go test ./daily/2026/06/26/worker_pool
```
