# 用白名单构建局部更新 SQL

后端接口做 PATCH 或局部更新时，最容易踩的坑是把前端传来的字段名直接拼进 SQL。这个示例使用“字段白名单 + 排序输出”的方式构建更新语句，只更新明确允许的列。

## 设计要点

- 业务字段与数据库列名分离，避免泄露表结构
- 未在白名单中的字段直接拒绝
- 生成 SQL 时按列名排序，便于测试断言和日志比对

## 示例说明

`Patch` 表示本次请求显式提供的字段；`BuildUpdate` 只把出现且允许的字段拼进 `SET` 子句，并把 `WHERE` 参数安全地追加到参数列表末尾。

## 运行方式

```bash
go test ./daily/2026/06/26/sql_patch_builder
```
