# 用 singleflight 思路避免缓存击穿

热点 key 过期瞬间，大量并发请求同时回源会把数据库或下游服务压垮。这个示例实现了一个极简的单飞缓存：同一个 key 在回源期间只允许一个 goroutine 执行加载，其他请求等待同一结果。

## 设计要点

- 先查缓存，命中直接返回
- 未命中时登记 inflight 请求，后续相同 key 复用同一加载结果
- TTL 到期后重新加载，避免长期返回陈旧数据

## 示例说明

`Cache.Get` 同时承担三层职责：读取缓存、合并并发 miss、在加载成功后写回缓存。这个模式很适合做本地热点保护层。

## 运行方式

```bash
go test ./daily/2026/06/26/cache_singleflight
```
