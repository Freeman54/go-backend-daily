# 用幂等键缓存重复请求响应

支付、下单、重试补偿这类接口往往需要幂等语义。这个示例实现了一个最小可用的内存幂等存储：第一次请求成功后把响应按幂等键缓存下来，后续相同键在 TTL 内直接返回缓存结果。

## 设计要点

- 幂等键一般来自请求头或业务主键组合
- 响应缓存必须带过期时间，避免永久占用内存
- 过期时即时删除，降低脏数据命中概率

## 示例说明

`Store.Remember` 在请求第一次成功后写入响应；`Store.Lookup` 在业务真正执行前查询是否已有结果，并在过期时完成清理。
