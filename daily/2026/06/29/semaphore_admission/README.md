# 用信号量做接口并发准入控制

高峰期接口最容易先被自身并发打垮。这个示例实现了一个极简准入闸门：只允许固定数量的请求同时进入业务逻辑，超过限制的请求等待空位，直到拿到令牌或被 `context` 取消。

## 设计要点

- 用带缓冲 channel 表示可用令牌
- `Acquire` 支持超时和主动取消，避免请求无限排队
- `Do` 统一封装获取令牌、执行业务、释放令牌

## 示例说明

这个模式适合放在热点接口、批处理入口或昂贵下游调用前面。相比无限制起 goroutine，它能把压力稳定在服务可承受范围内。

## 运行方式

```bash
go test ./daily/2026/06/29/semaphore_admission
```
