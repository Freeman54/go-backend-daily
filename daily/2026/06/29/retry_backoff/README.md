# 用可重试错误和退避策略保护下游

调用下游接口时，失败不一定都该立刻返回。超时、临时网络抖动、限流等错误往往适合重试；参数错误、鉴权失败则不该浪费重试次数。这个示例把“错误是否可重试”和“下一次等待多久”拆开建模。

## 设计要点

- 用 `RetryableError` 显式标记可重试错误
- 用 `Policy` 统一计算退避时间，便于测试和调参
- `Do` 在重试前检查 `context`，确保上游取消时及时退出

## 示例说明

实际业务里可以把 `sleep` 换成指数退避加抖动，也可以把最大重试次数按不同下游单独配置。

## 运行方式

```bash
go test ./daily/2026/06/29/retry_backoff
```
