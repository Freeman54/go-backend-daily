# 用优雅停机收敛后台任务

服务退出时如果直接杀进程，正在消费的消息、刷盘中的缓冲或尚未完成的定时任务都可能半途而废。这个示例实现了一个最小化的后台任务组：统一接收停止信号、等待 goroutine 收尾，并给停机过程设置上限。

## 设计要点

- 所有后台任务共享一个可取消的 `context`
- `Shutdown` 先广播取消，再等待全部 goroutine 退出
- 超过超时时间仍未退出时返回错误，便于上层记录告警

## 示例说明

这个模式适合 HTTP 服务里的异步刷新器、消息消费循环、定时同步任务等。它通常和 `signal.NotifyContext`、`http.Server.Shutdown` 组合使用。

## 运行方式

```bash
go test ./daily/2026/06/29/graceful_shutdown
```
