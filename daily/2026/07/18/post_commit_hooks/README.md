# 用 post-commit hooks 保证副作用只在事务成功后触发

事务里经常会顺手做一些副作用，例如发领域事件、刷新缓存、投递消息、写审计日志。如果这些动作在事务提交前就执行，后续一旦回滚，就会出现“主数据没落库，但副作用已经生效”的不一致。`post-commit hooks` 的核心就是把副作用先登记，只有提交成功才真正执行。

## 设计要点

- 在事务作用域内只注册 hook，不直接执行副作用
- `commit` 成功后顺序执行 hook，`rollback` 时统一丢弃
- hook 本身失败要单独暴露，便于后续补偿或告警

## 示例说明

示例里的 `Runner` 提供三种操作：

- `Add(hook)` 注册待执行动作
- `Commit()` 执行已登记 hook，并返回所有失败
- `Rollback()` 清空未提交的 hook

适合事务内先更新订单、库存、状态表，再在事务外触发缓存刷新、事件发布、审计补写。

## 运行方式

```bash
go test ./daily/2026/07/18/post_commit_hooks
```
