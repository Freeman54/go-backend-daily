# 用 priority admission 给高优请求预留并发名额

很多后端接口在高峰期会统一走到一个并发控制器里。如果普通流量先占满所有 worker，支付回调、写链路补偿、控制面接口这类关键请求就会一起排队。`priority admission` 的思路是在总并发上限之外，再为高优请求预留一部分容量。

## 设计要点

- 同时限制总并发，避免系统整体过载
- 给高优请求保留专用槽位，低优请求不能把它们提前占满
- 排队时优先唤醒高优请求，避免关键流量长期饥饿

## 示例说明

示例里的 `Limiter` 支持 `High` 和 `Low` 两档优先级：

- `Acquire(ctx, priority)` 获取执行资格
- 低优请求只能使用非保留槽位
- 高优请求既能使用保留槽位，也能使用公共空闲槽位
- 队列满时立即返回 `ErrQueueFull`

适合 RPC 网关、异步任务执行器、需要区分控制流量和业务流量的 worker 池。

## 运行方式

```bash
go test ./daily/2026/07/18/priority_admission
```
