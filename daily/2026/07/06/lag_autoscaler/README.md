# 用 lag autoscaler 按积压与吞吐估算消费者数量

消息消费组扩缩容如果只看 CPU 或实例数，经常会错过真正的背压信号。对队列型系统更有意义的做法，是把“当前积压量”“新消息流入速率”和“单消费者吞吐能力”合并起来，估算需要多少消费者才能在目标时间内追平积压，同时跟上新增流量。

## 设计要点

- 用目标追平时间把 `lag` 转换成所需消费速率
- 与实时流入速率取最大值，避免只追旧账却跟不上新消息
- 给扩缩容设置 `MaxScaleStep`，防止一次跳太大导致抖动

## 示例说明

示例中的 `Decide` 会根据策略和运行快照，计算建议消费者数量。它同时考虑积压追平目标、实时流入速率、单消费者吞吐和单次扩缩容步长。测试覆盖了按积压扩容、按流入速率稳态配置和零吞吐指标回退三种情况。

## 运行方式

```bash
go test ./daily/2026/07/06/lag_autoscaler
```
