# 用 savepoint 组织可嵌套的事务单元

一个服务方法里经常会复用“写库子流程”，但子流程既可能单独运行，也可能被更大的事务包裹。此时直接在子流程里无脑 `BEGIN/COMMIT` 会破坏外层事务边界；更稳妥的方案是在嵌套场景下使用 savepoint，把局部失败限制在子流程内部。

## 设计要点

- 外层单元负责真正的 `BEGIN/COMMIT/ROLLBACK`
- 内层单元只创建 `SAVEPOINT`，失败时回滚到局部检查点
- 局部失败后仍允许外层事务继续执行其他补偿或降级逻辑

## 示例说明

示例实现了一个纯内存 `Planner`，进入外层作用域时生成 `BEGIN`，进入内层作用域时生成递增 `SAVEPOINT`。退出时根据成功或失败返回对应 SQL 片段。测试覆盖了嵌套成功、内层局部回滚和外层整体回滚三个场景。

## 运行方式

```bash
go test ./daily/2026/07/06/savepoint_uow
```
