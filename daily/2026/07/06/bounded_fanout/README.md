# 用 bounded fanout 控制扇出并发与失败收敛

服务端常见的聚合接口会同时打多个下游，但“全部并行”很容易在流量放大时把线程、连接和配额吃满。更稳妥的做法是把 fanout 并发数控制在一个上限内，并在首个关键错误出现后尽快取消兄弟任务，避免无意义的尾部消耗。

## 设计要点

- 用有界信号量限制同时运行的 fanout 任务数量
- 保持结果顺序稳定，避免上层聚合逻辑再做额外重排
- 首个错误触发 `context` 取消，让还在运行的兄弟任务尽快退出

## 示例说明

示例中的 `Run` 会在 `maxParallel` 限制内执行一组任务，返回按输入顺序排列的结果；一旦任一任务失败，就取消上下文并返回首个错误。测试覆盖了顺序保持、并发上限和失败取消三种关键行为。

## 运行方式

```bash
go test ./daily/2026/07/06/bounded_fanout
```
