# 用 bulkhead executor 隔离不同依赖的并发风险

一个典型的 Go 后端同时依赖数据库、缓存、消息投递、邮件网关。如果某个依赖突然变慢，最坏情况不是它自己超时，而是把整个服务线程池和连接池都拖死。隔离舱（bulkhead）模式的目标就是把不同依赖的并发额度拆开，局部过载不要扩散成全局故障。

## 设计要点

- 每个 lane 独立配置并发上限
- 申请不到配额时，直接尊重调用方 `context` 超时或取消
- 暴露占用情况，便于接入指标或调试输出

## 示例说明

`Executor` 为每个依赖维护一个独立信号量。`Do` 在进入执行前先抢占该 lane 的 token，如果调用方 deadline 先到，就返回 `ErrLaneBusy` 和 `context` 错误。测试覆盖了 lane 之间互不影响，以及热点 lane 在配额耗尽时的失败行为。

## 运行方式

```bash
go test ./daily/2026/07/11/bulkhead_executor
```
