# 用日志采样预算避免相同错误刷爆告警与存储

线上故障经常不是“看不到日志”，而是“相同日志太多”。当数据库、缓存或者下游网关持续报错时，服务可能在几秒钟内打出成千上万条重复日志，既抬高存储成本，也掩盖真正需要关注的异常变化。一个实用做法是给每类错误建立固定时间窗内的采样预算。

## 设计要点

- 按错误 key 独立计数，避免一种热点错误吃掉所有日志额度
- 采用固定窗口预算，超过阈值后静默丢弃重复日志
- 保持实现简单，方便挂到 logger hook 或 middleware 前面

## 示例说明

`Sampler` 维护 `key -> 时间窗计数` 状态，`Allow` 用来判断当前日志是否应该保留。测试覆盖单 key 限流、时间窗重置，以及不同错误 key 预算互不影响。

## 运行方式

```bash
go test ./daily/2026/07/11/log_sampling_budget
```
