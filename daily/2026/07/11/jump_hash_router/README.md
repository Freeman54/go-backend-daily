# 用 jump hash 路由消息分区并减少扩容迁移

消息队列、分片缓存、分库分表都会遇到同一个问题：如何把同一个 key 稳定地路由到固定分区，同时在分区数量变化时尽量减少迁移量。普通取模虽然简单，但从 8 个分区扩容到 9 个分区时会大面积重映射。Jump Consistent Hash 更适合这种在线扩容场景。

## 设计要点

- 同一个业务 key 始终得到稳定分区
- 分区数增加时，只迁移一小部分 key
- 算法不依赖额外元数据，适合在生产路由层直接使用

## 示例说明

`Router` 基于 FNV-1a 和 jump hash 计算分区编号，`MovedRatio` 用于评估扩容前后的 key 迁移比例。测试覆盖路由稳定性、从 8 扩到 9 分区时的低迁移特性，以及非法分区配置的降级行为。

## 运行方式

