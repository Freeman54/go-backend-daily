# 用 after commit hooks 延迟执行事务外副作用

数据库事务里最适合做的是状态变更，不适合直接塞邮件发送、消息发布、缓存刷新这类外部副作用。因为一旦副作用已经发出而事务回滚，系统就会出现明显的不一致。一个常见改法是先把副作用登记为提交后钩子，确认事务成功后再统一执行。

## 设计要点

- 事务提交前只登记 hook，不直接触发外部动作
- 提交完成后按注册顺序执行，便于控制 outbox、缓存、审计等依赖关系
- 多个 hook 失败时使用 `errors.Join` 保留完整排障信息

## 示例说明

`Unit` 提供 `OnCommit` 和 `Commit` 两个方法，模拟服务层在事务内记录提交后动作。测试覆盖顺序执行、多个错误合并，以及事务完成后拒绝继续注册 hook 的保护逻辑。

## 运行方式

```bash
go test ./daily/2026/07/11/after_commit_hooks
```
