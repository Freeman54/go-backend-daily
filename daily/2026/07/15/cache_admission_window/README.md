# 用 admission window 避免一次性冷数据挤占缓存

不是所有数据都值得进缓存。后台系统里经常会遇到“只访问一次的冷 key”，如果首次命中就写入缓存，很容易把真正的热点数据挤掉。`admission window` 的思路是先观察短时间内同一 key 的访问频次，达到阈值再允许写入缓存。

## 设计要点

- 为每个 key 维护一个短时间窗口内的访问计数
- 只有计数达到阈值，才认为它值得占用缓存容量
- 超出窗口后自动重新计数，避免历史热点永久占优

## 示例说明

示例实现了一个最小化的准入策略，可以挂在 read-through cache 之前，先判断当前 key 是否应该被缓存。测试验证：

- 同一 key 在窗口内达到阈值后才准入
- 窗口过期后需要重新累计
- 不同 key 的计数相互独立

## 运行方式

```bash
go test ./daily/2026/07/15/cache_admission_window
```
