# 用请求指纹稳定识别幂等请求

幂等键如果只靠客户端随手传一个字符串，服务端很难判断“同一个 key 是否对应同一份业务语义”。更稳妥的做法是在落库前把方法、路径、查询参数、关键请求头和请求体归一化，再生成稳定指纹，用它校验重复请求是否真的等价。

## 设计要点

- 对 method、path、query、headers、body 做规范化拼接，消除 map 遍历顺序带来的不稳定
- header 名统一按大小写无关处理，query value 先排序再串接
- 可以把结果和客户端传入的幂等键一起存储，重复请求到来时先比对指纹

## 示例说明

示例提供 `CanonicalString` 和 `Sum` 两个函数，适合在支付、创建工单、批量导入等“不能重复执行”的接口入口使用。测试验证：

- 参数顺序不同但业务语义一致时，指纹保持稳定
- 请求体变化时，指纹必须发生变化

## 运行方式

```bash
go test ./daily/2026/07/15/idempotency_fingerprint
```
