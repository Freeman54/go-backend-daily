# 用 poison message tracker 尽早隔离必然失败的消息

消息系统里的失败并不都值得重试。超时、瞬时限流可以稍后再试，但字段缺失、版本不兼容、签名错误这类“毒消息”即使重投 100 次也不会成功。把这两类失败混在一起，只会放大积压和告警噪音。

## 设计要点

- 已知不可恢复错误码直接隔离到 quarantine 或人工处理队列
- 普通瞬时错误允许在阈值内重试，超过阈值再隔离
- 消息成功后清空失败计数，避免下一次处理被历史状态污染

## 示例说明

示例实现了一个轻量级消息失败决策器，输入消息 ID 和错误码，输出 `retry`、`ack` 或 `quarantine`。测试覆盖：

- 普通错误按次数阈值重试
- 已知 poison code 立即隔离
- 后续成功后会清空累计失败次数

## 运行方式

```bash
go test ./daily/2026/07/15/poison_message_tracker
```
