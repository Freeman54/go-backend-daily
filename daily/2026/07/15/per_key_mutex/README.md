# 用 per-key mutex 串行化同一资源的并发更新

订单、库存、额度等资源往往允许不同 key 并发处理，但同一个 key 的更新必须串行，否则很容易出现覆盖写、重复扣减或状态机乱序。`per-key mutex` 的思路是给每个业务 key 动态分配一把锁，只阻塞同 key 请求。

## 设计要点

- `Locker.Lock(key)` 只会阻塞相同 key 的并发调用
- 锁条目带引用计数，最后一个请求释放后自动清理，避免 map 长期膨胀
- 不同 key 不共享同一把锁，能保住整体吞吐

## 示例说明

示例实现了一个轻量级 key 锁管理器，适合包在账户变更、库存预扣或幂等写入入口处。测试覆盖三件事：

- 同 key 请求必须串行
- 不同 key 可以并行
- 请求结束后要把空闲锁从索引中移除

## 运行方式

```bash
go test ./daily/2026/07/15/per_key_mutex
```
