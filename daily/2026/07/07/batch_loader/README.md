# 用 batch loader 合并热点读，降低数据库 N+1

列表页或聚合接口经常会在一次请求里反复按 ID 查库，如果每个 ID 都单独访问存储层，就会出现典型的 N+1 查询问题。一个轻量的 batch loader 可以在很短的时间窗内把重复 key 聚合起来，统一批量查询，再把结果分发回各个调用方。

## 设计要点

- 用短暂等待窗口收集同一批次内的 key，兼顾吞吐与尾延迟
- 对重复 key 做去重，避免一批请求里对数据库重复命中
- 在批量数达到阈值时立即 flush，避免单批过大影响响应时间

## 示例说明

示例中的 `Loader` 负责收集 `Load` 请求，并在定时器触发或达到 `maxBatch` 后调用一次 `fetch`。测试覆盖了批次内 key 去重、达到批量阈值时立即刷新，以及底层查询错误向所有等待者传播这三类后端常见行为。

## 运行方式

```bash
go test ./daily/2026/07/07/batch_loader
```
