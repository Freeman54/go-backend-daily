# 用 slow query sampler 控制日志成本

数据库访问日志是排查线上问题的重要依据，但如果把所有成功请求都完整打印，磁盘、带宽和日志索引成本会迅速膨胀。更稳妥的做法是对“失败”和“慢查询”全量保留，而对普通成功请求做稳定采样。

## 设计要点

- 失败请求必须全量记录，不能被采样丢掉
- 超过阈值的慢查询必须全量记录，方便做根因分析
- 正常成功请求使用稳定采样，同一 SQL 模板的行为保持一致，便于比较

## 示例说明

示例中的 `Sampler` 采用固定阈值判断慢查询，对失败或慢请求直接放行，对普通请求则用操作名做哈希采样。测试覆盖了失败与慢查询强制记录、快速成功请求稳定采样，以及非法配置校验。

## 运行方式

```bash
go test ./daily/2026/07/07/slow_query_sampler
```
