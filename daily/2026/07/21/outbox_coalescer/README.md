# 用 outbox coalescer 减少重复领域事件发布

事务外盒模式能解决“数据库提交成功但消息没发出去”的一致性问题，但如果同一个聚合在短时间内被多次修改，外盒表里很容易堆出很多语义上已经过时的事件。消费者虽然最终能收敛，但链路成本、重放压力和排障噪音都会明显上升。

这个示例实现了一个简单的 `outbox coalescer`：

- 以聚合 ID 为维度，只保留最新版本的事件
- `Flush()` 时按聚合 ID 排序输出，便于稳定测试和批量发布
- 适合在真正写入 outbox 表之前做一次进程内压缩

适合：

- 同一订单、同一账户在一次事务窗口里发生多次状态变化
- 批量同步任务需要只发布最终态，而不是中间态
- 事件量很大，希望先做一次 cheap compaction 再落库

运行：

```bash
go test ./daily/2026/07/21/outbox_coalescer
```
