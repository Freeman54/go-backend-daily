# 用 error signal map 统一错误处理和观测标签

很多 Go 服务都会在 handler、service、DAO 三层分别判断错误，最后造成日志字段、监控指标、重试策略都不一致。更实用的做法是先把错误归类成稳定的信号，再让日志、指标和上层协议统一消费这些信号。

这个示例实现了一个轻量级映射器：

- 用 `errors.Is` 识别包装后的业务错误和依赖错误
- 输出统一的 `Code`、日志 `Level`、`Retryable` 标志和指标名
- 让 handler 可以基于 `Signal` 决定 HTTP 状态码、报警级别或重试行为

适合：

- API 层做统一错误协议映射
- 后台任务把错误分流到重试队列或死信队列
- 监控侧按错误类别聚合，而不是按字符串模糊匹配

运行：

```bash
go test ./daily/2026/07/21/error_signal_map
```
