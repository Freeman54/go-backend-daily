# 用公平 worker 队列避免大租户饿死小租户

很多批处理、异步任务和消息消费逻辑默认按 FIFO 拉取任务，但当某个大租户持续灌入大量任务时，其他租户很容易长期得不到执行机会。公平队列的核心目标不是“绝对平均”，而是在吞吐和隔离之间取得可预期的平衡。

这个示例实现了一个按租户轮转的内存队列：

- `Enqueue(tenant, value)` 把任务放到租户自己的子队列
- `Next()` 以 round-robin 的方式在不同租户之间轮转取任务
- `Len()` 返回剩余总任务数，便于和 worker 池、监控指标结合

这种模式适合：

- 多租户异步任务执行
- 分区消费者在应用层做二次公平调度
- 内部批处理系统防止热点租户长期占满 worker

运行：

```bash
go test ./daily/2026/07/21/fair_worker_queue
```
