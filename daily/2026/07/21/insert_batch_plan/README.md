# 用 insert batch plan 控制批量写库的行数和参数上限

批量写库通常不是“越大越好”。很多数据库或驱动都会对单条 SQL 的参数数量、包体大小、锁持有时间产生隐性上限。如果应用层不提前拆批，线上经常会遇到偶发的 SQL 太长、Prepared 参数过多，或者大事务把尾延迟拖高的问题。

这个示例实现了一个简单的拆批规划器：

- 输入总行数、列数、单批最大行数、单批最大参数数
- 自动取更严格的限制，生成每一批的 `[start, end)` 区间
- 让写入层专注于拼接 SQL，本身不必再判断拆批策略

适合：

- 批量 `INSERT` / `UPSERT`
- outbox、审计日志、指标落库等高频写入
- 希望在应用层把事务大小控制在稳定区间的场景

运行：

```bash
go test ./daily/2026/07/21/insert_batch_plan
```
