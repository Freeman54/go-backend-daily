# 用 stage budget 拆分单次请求的超时预算

后端请求常常会串行经过参数校验、下游查询、数据库写入、审计上报等多个阶段。如果直接把整条链路挂在一个总超时上，前面的阶段一旦拖慢，后面的关键写入阶段就会被被动压缩，最后只留下“超时了”这一种粗粒度结论。

这个示例实现了一个简单的 `stage budget planner`：

- 从请求 `context` 的 deadline 中取出总剩余时间
- 预留一段 `reserve`，避免把预算全部吃光
- 按权重把剩余时间分配给每个处理阶段

适合用于：

- HTTP handler 内部的串行调用链预算拆分
- 任务执行器里把剩余超时分给准备、执行、回收三个阶段
- 下游依赖很多、需要显式控制慢阶段影响面的服务

运行：

```bash
go test ./daily/2026/07/21/stage_budget
```
