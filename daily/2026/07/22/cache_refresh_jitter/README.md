# 用稳定抖动打散缓存集中刷新

大量缓存若使用相同 TTL，会在部署或批量预热后同时过期，瞬间把数据库打满。刷新抖动通过给每个 key 分配略有差异的刷新时间，把负载摊到更宽的窗口。

`RefreshAfter` 对 key 做稳定哈希，在 `base ± percent` 范围内生成确定性刷新间隔。稳定性意味着同一个 key 在不同进程中得到相同结果，既能打散流量，也不会因每次读取随机变化而难以排障。生产中可把它用于后台 refresh-ahead，并保留硬 TTL 作为兜底。

```bash
go test ./daily/2026/07/22/cache_refresh_jitter
```
