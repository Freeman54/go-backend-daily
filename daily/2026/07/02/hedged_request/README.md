# 用 hedged request 降低尾延迟

当主请求偶发抖动时，可以在短暂等待后再发起一个“备份请求”。只要任意一个成功，就立刻取消另一个，从而用少量额外流量换更稳的尾延迟。

## 设计要点

- 主请求先发，超过阈值才启动 hedge，避免无意义放大流量
- 任意一个请求成功后立即取消另一个，减少资源浪费
- 两个请求都失败时返回聚合错误，便于记录诊断信息

## 示例说明

这个模式适合读接口、搜索、只读副本查询等对延迟敏感的路径。实践里通常只对高延迟分位上的请求启用，而不是所有请求都双发。

## 运行方式

```bash
go test ./daily/2026/07/02/hedged_request
```
