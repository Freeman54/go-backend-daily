# 用 burn rate 判断错误预算告警

只盯瞬时错误率很容易误报，尤其是低流量服务。SRE 更常用的方法是看错误预算消耗速度，也就是 burn rate，再同时检查短窗口和长窗口，区分“尖峰抖动”和“持续故障”。

## 设计要点

- 先把窗口错误率换算成相对错误预算的消耗倍数
- 短窗口负责快速发现问题，长窗口负责过滤瞬时抖动
- 只有两个窗口都超过阈值时才告警，降低误报

## 示例说明

生产环境里常见配置是把这个判断接到 Prometheus 录制规则或告警规则中，例如针对 `99.9%` 可用性定义不同窗口与阈值组合，分别触发 page 和 ticket。

## 运行方式

```bash
go test ./daily/2026/07/02/slo_burnrate
```
