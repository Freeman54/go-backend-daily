# 用 TTL 抖动分散缓存集中失效

如果一批热点 key 在同一时刻写入且 TTL 完全相同，它们很可能在同一秒一起过期，形成瞬时回源尖峰。这个示例在基础 TTL 上增加随机抖动，把过期时间自然摊开。

## 设计要点

- 保持统一的基础 TTL，方便理解缓存新鲜度
- 额外叠加一个可控范围内的随机抖动，打散回源时间
- 抖动范围不宜过大，否则会让数据新鲜度不可预期

## 示例说明

实战里可以把这个策略用于热点列表、配置快照、用户画像等批量写入缓存的场景，通常与 singleflight、限流或本地缓存一起使用效果更稳。

## 运行方式

```bash
go test ./daily/2026/07/02/ttl_jitter_cache
```
