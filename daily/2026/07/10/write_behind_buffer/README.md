# 用 write-behind buffer 合并热点写入

热点计数、异步审计、批量状态落库都适合先在内存里合并一小段时间，再批量刷向存储层。这样能明显减少数据库或缓存的写放大，但前提是刷新边界要可控，进程退出时也不能丢最后一批。

## 设计要点

- 相同 key 的变更先在内存中聚合，降低下游写次数
- 用“键数量阈值 + 定时 flush”双触发，兼顾吞吐和时效
- 退出时主动 flush，避免 write-behind 模式吞掉尾批数据

## 示例说明

示例里的 `Buffer` 把多个 `Mutation` 聚合成批量 map，再交给 `onFlush` 回调统一落库。测试覆盖相同 key 合并、周期刷新，以及服务关闭时的最终刷新。
