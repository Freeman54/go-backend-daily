# 用 priority limiter 给关键流量预留并发额度

同一个服务里常常同时承载用户请求、后台任务和补偿作业。如果所有流量共享同一个并发池，低优先级任务很容易把关键请求的容量挤占掉。一个简单可落地的策略，是给高优先级流量预留一部分并发额度。

## 设计要点

- 总并发上限控制整体资源消耗
- 低优先级只允许使用非预留额度，避免把高优先级饿死
- 高优先级既能使用预留额度，也能使用当前未占用的公共额度

## 示例说明

示例里的 `Limiter` 维护高低优先级的占用计数。测试覆盖低优先级无法侵占预留配额、高优先级可以消费保留容量，以及释放后容量能重新被占用。

## 运行方式

```bash
go test ./daily/2026/07/10/priority_limiter
```
