# 用 context handoff 预留尾部预算给下游和清理逻辑

父请求带着统一 deadline 进入服务后，业务代码常常会把整个剩余时间一股脑交给下游调用，导致尾部清理、日志刷盘或兜底缓存根本没有预算。更稳妥的做法是先预留一段尾部时间，再按权重把剩余预算切给不同阶段。

## 设计要点

- 统一从 `context.Context` 读取 deadline，避免手写多个超时常量互相打架
- 为清理、降级或响应编码预留 `reserve`，不要把最后几毫秒全部透支给下游
- 不同阶段按权重切片，同时保证每一段至少有可工作的最小预算

## 示例说明

示例实现了一个 `Plan` 函数：从 context deadline 计算可分配时间，扣掉尾部预留后，再按阶段权重分配预算。测试覆盖了权重分配、缺失 deadline 和预算不足三种场景。

## 运行方式

```bash
go test ./daily/2026/07/05/context_handoff
```
