# 用 errors.Join 做并发错误聚合后的处置决策

并发 fan-out 或批处理任务里，经常会同时拿到多个错误：有些适合重试，有些属于输入脏数据必须立即失败，还有些只是旁路埋点失败可以忽略。把这些错误直接拼成字符串没有后续价值，更实用的做法是保留错误语义，再统一得出处置决策。

## 设计要点

- 并发任务返回的多个错误先用 `errors.Join` 聚合，不丢失原始因果链
- 错误需要带上可判定的业务语义，例如 `retryable`、`fatal`、`ignored`
- 决策优先级要稳定明确，通常是 `fatal > retryable > ignored`

## 示例说明

示例提供了三种带语义的错误包装器，并实现 `Decide` 函数扫描聚合错误。测试覆盖了致命错误优先、仅可重试错误、以及可忽略错误三类情况。
