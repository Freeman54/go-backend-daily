# 消费端序列守卫：识别重复与消息缺口

消息系统通常保证“至少一次”投递，因此消费者必须能识别重复；分区键配置错误、生产端并发或补偿任务还可能造成同一业务实体的消息乱序。只保存最后处理的序列号，可以同时识别这两类问题。

`Guard` 按业务 key 记录最后序列号：小于等于该值视为重复，恰好加一才应用，更大的值返回 `ErrSequenceGap` 且不推进状态。调用方发现缺口后可暂停该 key、重拉缺失事件或进入人工补偿，不能直接跳过。

示例是进程内版本。生产系统应在数据库事务中把业务变更与序列号推进原子提交，或者使用具备条件写能力的状态存储。

```bash
go test ./daily/2026/07/23/message_sequence_guard
```
