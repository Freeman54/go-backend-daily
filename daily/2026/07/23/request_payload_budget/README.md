# 请求载荷预算：在解码前限制读取量

HTTP 服务若直接对请求体调用 `io.ReadAll` 或让解码器无限读取，攻击者可以用超大载荷持续占用内存和连接。仅检查 `Content-Length` 也不可靠，因为分块传输可能没有该字段，客户端还可能谎报长度。

`Read` 用 `io.LimitReader` 最多读取“预算 + 1”字节：多出的一个字节用于可靠判断请求是否越界，同时避免把整个超大载荷载入内存。源读取错误会保留错误链，越界则返回稳定的 `ErrPayloadTooLarge`，HTTP 层可将其映射为 413。

生产环境应在网关、HTTP handler 和具体字段规则三个层级分别设限；载荷上限也要与解压后的大小限制配合，防止压缩炸弹。
