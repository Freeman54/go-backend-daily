# 用代际号阻止失效后的旧值回填缓存

Cache-Aside 中可能出现竞态：读请求缓存未命中并开始查数据库，随后写请求更新数据库并删除缓存，但较慢的读请求最后又把旧值写回缓存。仅靠删除无法识别这次过期回填。

`Guard` 在读取开始时返回 key 当前的代际号。失效操作递增代际号；回填只有在代际号仍一致时才能提交，因此失效前启动的慢查询会被拒绝。示例还对字节切片进行复制，防止调用方绕过锁修改缓存内容。

单机实现适合展示算法；分布式场景需要把代际号放进 Redis、数据库版本列或支持比较交换的存储中。

```bash
go test ./daily/2026/07/23/cache_generation_guard
```
