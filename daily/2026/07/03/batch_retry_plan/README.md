# 用批量重试计划拆分成功、重试和死信消息

消息队列批量消费时，常见问题不是“整批成功”或“整批失败”，而是同一批消息里既有临时失败，也有永久失败。如果不先做结果拆分，就会把本可确认的消息一并重试，放大重复消费和积压。

## 设计要点

- 区分成功、可重试失败、不可重试失败三类结果
- 可重试消息只在剩余重试预算内回队，超出预算后直接转死信
- 先形成明确计划，再驱动 ack、重试发布和死信投递，便于指标统计和审计

## 示例说明

示例把一批消费结果转换成 `AckIDs`、`RetryIDs`、`DLQIDs` 三组输出。这样在真实系统里可以很自然地衔接 broker ack、延迟重试 topic 和死信队列。
