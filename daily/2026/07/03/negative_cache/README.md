# 用 negative cache 缓解缓存穿透

当热点 key 不存在时，如果每次都直接打到数据库，接口会被重复的“查无此人”请求拖垮。negative cache 的做法是把“不存在”也短暂缓存起来，用更短 TTL 挡住瞬时穿透流量。

## 设计要点

- 正常数据和“不存在”结果使用不同 TTL，避免把临时空洞缓存太久
- 读命中 negative cache 时直接返回，不再访问后端存储
- 一旦真实数据出现，必须能覆盖之前的 not found 记录

## 示例说明

示例实现了一个内存缓存，支持正向命中、负向命中和过期淘汰。测试覆盖了双 TTL 生效以及 not found 被后续写入覆盖的场景。

## 运行方式

```bash
go test ./daily/2026/07/03/negative_cache
```
