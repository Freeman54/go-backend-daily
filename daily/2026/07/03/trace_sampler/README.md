# 用分层 trace sampling 保住异常与慢请求

全量保留 tracing 往往成本太高，但如果只做简单随机采样，又容易把真正有价值的异常链路丢掉。更实用的策略是分层采样：正常请求按低比例抽样，错误请求、慢请求和人工指定流量则强制保留。

## 设计要点

- 错误请求和慢请求优先保留，避免诊断信息被随机丢弃
- 基础采样使用稳定哈希，保证同一 trace ID 多次判定结果一致
- 采样决策最好附带原因，便于在指标面板里解释样本来源

## 示例说明

示例实现了一个简单采样器，支持基础千分比采样，同时对 `forced`、`error`、`slow` 三类请求直接保留。测试覆盖了优先级和稳定性。
