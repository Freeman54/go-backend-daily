# 用 saga compensation 兜住跨服务失败回滚

本地事务跨不出单个数据库，到了“扣库存、扣余额、发通知”这类跨服务流程，就需要用 saga 把长事务拆成一串本地动作，并为每一步准备补偿逻辑。这样即使中途失败，也能把前面已经完成的动作尽量回滚到一致状态。

## 设计要点

- 每个步骤只做本地可控的原子操作，失败时靠补偿撤销已完成动作
- 补偿顺序必须与执行顺序相反，避免留下新的依赖残留
- 原始失败和补偿失败都要保留，方便排查半成功链路

## 示例说明

示例实现了一个最小化 saga 执行器：按顺序执行步骤，一旦出错就逆序调用补偿函数，并把原始错误与补偿错误拼接返回。它适合类比订单创建、营销发券、资源开通等流程编排。
