# 用时间窗去重降低重复指标和告警噪音

服务雪崩、依赖抖动或批量失败时，同一类日志、指标或告警可能在几秒内被刷出成百上千次。信号并没有增加，噪音却会挤占观测系统吞吐。时间窗去重的核心思路是：同一个 key 在窗口期内只放行一次，把剩余重复事件折叠掉。

## 设计要点

- 用 `key -> last emit time` 记录最近一次放行时间
- 窗口期内的重复事件直接丢弃，窗口外重新放行
- 按 key 独立判断，避免不同告警之间互相影响

## 示例说明

示例提供 `Deduper.ShouldEmit(key, now)`，适合：

- 限制相同错误标签的自定义指标上报频率
- 折叠短时间内重复的告警触发
- 控制热点路径日志和事件风暴

测试覆盖同 key 窗口抑制和多 key 独立放行。

## 运行方式

```bash
go test ./daily/2026/07/17/metric_window_dedup
```
