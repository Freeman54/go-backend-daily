# 用 Retry-After hint 把限流恢复时间明确告诉客户端

当接口因为限流、维护窗口或下游熔断而暂时拒绝请求时，只返回 `429` 或 `503` 往往不够。客户端如果不知道多久后再试，就可能立刻重试，继续放大压力。更好的做法是输出 `Retry-After` 提示，把服务端预计恢复时间转换成稳定的秒数。

## 设计要点

- 把未来时刻 `retryAt` 转成客户端更容易消费的秒数
- 对不足 1 秒的剩余时间向上取整，避免返回 `0`
- 对过大的等待时间设置上限，防止把异常状态无限放大给调用方

## 示例说明

示例提供 `Seconds(now, retryAt, maxSeconds)`，适合：

- 网关限流后返回 `Retry-After`
- 运维摘流期间给调用方明确等待窗口
- 熔断器半开前给重试客户端一个统一节奏

测试覆盖向上取整、最小值保护和上限截断。

## 运行方式

```bash
go test ./daily/2026/07/17/retry_after_hint
```
