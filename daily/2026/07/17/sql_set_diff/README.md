# 用 set diff 生成关联表增删计划

后端接口里经常会收到“把用户绑定到这些角色”“把商品挂到这些分类”这类整集合更新请求。直接先删后插会放大数据库写入，也容易破坏审计和变更语义。更稳妥的做法是先对比当前集合和目标集合，只生成真正需要执行的 `INSERT` 与 `DELETE`。

## 设计要点

- 先把当前值和目标值归一成 set，消除重复输入
- 分别计算“目标里有但当前没有”的待插入集合
- 计算“当前有但目标没有”的待删除集合
- 输出排序后的结果，便于测试、日志和后续批量 SQL 生成

## 示例说明

示例提供 `Build(current, desired)`，返回：

- `ToInsert`：应新增的关联 ID
- `ToDelete`：应删除的关联 ID

适合角色绑定、标签同步、多对多关系补丁更新等场景。业务层可以基于这个结果继续组装 `INSERT ... ON CONFLICT DO NOTHING` 或批量删除语句。

## 运行方式

```bash
go test ./daily/2026/07/17/sql_set_diff
```
