# 用 tenant fair queue 避免热点租户长期饿死其他请求

多租户后端常见的问题不是总吞吐不够，而是一个热点租户在短时间灌入大量任务后，把其他租户的请求都压到队尾。对任务队列做简单的轮转公平调度，能够在不引入复杂优先级系统的前提下，把延迟更均匀地分配给不同租户。

## 设计要点

- 每个租户维护独立 bucket，避免热点租户独占全局 FIFO
- 出队时按活跃租户轮转，保证冷租户也能持续获得处理机会
- 某个租户 bucket 清空后，立即从活跃集合移除，减少空转

## 示例说明

`Queue` 支持按租户入队与公平出队。测试覆盖热点租户和冷租户交替消费、空 bucket 移除，以及空队列行为。
