# 用 batch flusher 把零散写请求合并成更稳定的批量落库

数据库、日志系统或消息网关经常不怕“总量大”，而怕“每次都很小”。如果业务高频提交单条写入，不仅连接和事务开销高，还容易把下游打成抖动尖峰。一个常见优化是按条数或时间窗把零散请求聚合成批次，再统一刷盘。

## 设计要点

- 达到批量阈值时立即刷新，降低单次写入开销
- 即使流量不高，也要在最大等待时间到达后主动刷新，避免小批次长期滞留
- 刷盘失败时保留当前 batch，给调用方重试机会

## 示例说明

`Flusher` 维护待刷新的内存 batch，支持按条数触发、按等待时间触发，以及关闭时兜底刷新。测试覆盖批量阈值刷新、时间窗刷新和失败后保留 batch。

## 运行方式

```bash
go test ./daily/2026/07/12/batch_flusher
```
