# 用 write-through cache 保证缓存和主存写入顺序一致

后端里最容易失控的缓存问题之一，是主存和缓存分别写入时的时序错乱。比如旧请求比新请求晚到，却把缓存回写成旧值，导致读流量持续命中过期内容。一个简单实用的办法是给写入附带版本号，在主存和缓存两层都执行版本保护。

## 设计要点

- 先写主存，再写缓存，保证缓存只反映已经持久化的数据
- 用版本号拒绝过期写入，避免慢请求覆盖新值
- 读取时支持从主存回填缓存，减少缓存冷启动期间的重复 miss

## 示例说明

`Store` 组合内存版 repo 和 cache，实现版本保护的 write-through 语义。测试覆盖双写成功、旧版本写入被拒绝，以及缓存 miss 后从主存回填。

## 运行方式

```bash
go test ./daily/2026/07/12/cache_write_through
```
