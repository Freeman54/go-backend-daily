# 用 health snapshot 把依赖状态聚合成可执行的就绪结论

很多服务的健康检查不是“单个依赖好不好”，而是“当前依赖组合是否还能继续接流量”。例如搜索故障时，接口也许还能降级返回；但主库故障时，实例就应该立刻摘流。把多个依赖探针聚合成统一快照，有助于 readiness、告警和运维动作保持一致。

## 设计要点

- 区分关键依赖与非关键依赖，避免所有失败一刀切摘流
- 输出 `ready`、`degraded`、`unready` 三种模式，方便接到探针或控制面
- 聚合统计健康、降级和故障数量，便于日志和指标消费

## 示例说明

`Build` 接收多个依赖状态，给出整体就绪结论。测试覆盖全部健康、非关键依赖故障导致降级，以及关键依赖故障触发摘流。

## 运行方式

```bash
go test ./daily/2026/07/12/health_snapshot
```
