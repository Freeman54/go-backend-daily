# 用原子快照安全热更新运行时配置

线上配置热更新常见的坑，不是“能不能改”，而是“改的过程中会不会读到半成品”。例如一边更新超时，一边更新连接池大小，如果多个 goroutine 分别读到新旧混合配置，就很容易出现偶发故障。比较稳的方式是把整份配置做成不可变快照，更新时整包替换。

## 设计要点

- 加载配置时返回不可变快照，避免调用方拿到共享可变状态
- `Reload` 先校验，再整体替换，失败时保留旧版本继续服务
- 版本号随每次热更新递增，方便日志和指标定位配置生效批次

## 示例说明

`Manager` 用 `atomic.Value` 保存当前配置快照，对外暴露 `Load` 和 `Reload`。测试覆盖非法配置、版本递增，以及读取结果的防御性拷贝。

## 运行方式

```bash
go test ./daily/2026/07/13/config_snapshot
```
