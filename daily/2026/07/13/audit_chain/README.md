# 用 audit chain 构建可校验的操作审计轨迹

很多后台系统都会记审计日志，但“有日志”不等于“日志可信”。如果有人能修改中间某一条操作记录，而下游又只看最终文本，就很难发现历史是否被篡改。一个轻量办法是让每条审计事件都串上前一条的摘要，形成可校验的链式轨迹。

## 设计要点

- 每条事件的摘要都包含前一条摘要，任意中间修改都会破坏整条链
- 审计事件保留操作者、动作、资源和时间，便于追溯
- 校验逻辑独立成 `VerifyEvents`，适合在离线核对或导出文件时复用

## 示例说明

`Trail` 负责顺序追加事件，`VerifyEvents` 负责回放校验。测试覆盖正常校验通过，以及篡改动作字段后链路校验失败。

## 运行方式

```bash
go test ./daily/2026/07/13/audit_chain
```
