# 用稳定哈希做可回放的 canary 发布

真实发布里最怕的问题不是有没有灰度，而是灰度结果不稳定。今天命中、明天没命中，会让指标对比、问题复现和回滚分析都变得困难。更稳妥的做法，是基于特征开关 key 和 actor 标识做稳定哈希，让同一个用户或租户始终落在固定桶位。

## 设计要点

- 用稳定哈希把 actor 映射到 `0-99` 的桶位，百分比灰度天然可回放
- `denylist` 优先级高于 `allowlist`，先保守兜底，再局部放量
- `100%` 和 `0%` 两个边界明确返回，避免隐式行为

## 示例说明

`Enabled` 会根据规则和 actor ID 计算是否命中灰度。测试覆盖名单优先级、稳定性，以及百分比边界行为。

## 运行方式
