# 用 fallback budget 限制降级结果的放大效应

降级本身不是坏事，但“无限制地降级成功”会掩盖故障规模。比如一个依赖持续超时，服务端全部回退到兜底数据，表面上成功率还不错，实际上用户拿到的是越来越陈旧或越来越粗糙的结果。`fallback budget` 的作用，就是限制单位时间内允许返回多少次降级结果，超额后宁可显式失败，也不要让问题长期失真。

## 设计要点

- 把降级视为一种有限预算，而不是永远安全的默认路径
- 用固定时间窗统计降级次数，窗口滚动后自动恢复预算
- 同时暴露剩余额度，便于打点到指标系统里做告警

## 示例说明

`Budget` 维护某个时间窗内已经使用的降级次数。测试覆盖预算耗尽、时间窗重置，以及剩余额度的计算。

## 运行方式

```bash
go test ./daily/2026/07/13/fallback_budget
```
