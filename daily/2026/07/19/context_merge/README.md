# 用 context merge 汇总多个上游取消信号

聚合任务、旁路写、双写校验这类场景，往往会同时受到多个上游 `context` 影响：请求上下文取消了要停，调度器窗口结束了也要停，兜底超时到了同样要停。只沿用其中一个 `context`，很容易漏掉别的终止条件。

`context merge` 的目标是把多个父上下文合并成一个子上下文，只要任一父上下文结束，合并后的上下文就结束，并继承最早 deadline。

## 设计要点

- 选择多个父上下文里的最早 deadline
- 任一父上下文取消时，合并上下文立刻结束
- 保留取消原因，便于日志和指标上报

## 示例说明

示例实现提供 `Merge(parents...)`：

- 返回新的合并上下文和显式取消函数
- 父上下文只要有一个结束，就会把原因传递给结果上下文
- 没有 deadline 时也能工作

适合聚合查询、双写链路、跨组件协同任务等场景。
