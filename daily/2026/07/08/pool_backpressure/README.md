# 用连接池背压控制保护数据库

数据库连接池被打满时，问题往往不只出在数据库本身，还出在应用层继续放量，把等待队列越堆越长。一个简单的连接池背压控制器可以根据池使用率和等待请求数做三段式决策：正常放行、降级处理、直接拒绝。

## 设计要点

- 使用率接近上限时优先触发降级，例如改走缓存、只返回摘要或关闭昂贵查询
- 等待队列爆炸时尽早拒绝，避免请求在应用层长时间堆积
- 阈值需要和 `db.Stats()`、接口 SLA 以及调用方重试策略一起校准

## 示例说明

示例中的 `Controller` 输入是一个简化版连接池快照，输出三种决策：`admit`、`degrade`、`reject`。测试覆盖健康、接近饱和和等待队列失控三种后端常见状态。
