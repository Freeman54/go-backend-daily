# 用安全排序白名单构建列表接口 ORDER BY

列表接口常见需求是支持按时间、金额、状态等字段排序。如果直接把用户输入拼进 SQL，轻则生成慢查询，重则引入 SQL 注入风险。更稳妥的做法是用白名单把外部字段映射到内部列名，并固定补上稳定的二级排序。

## 设计要点

- 对外暴露“业务字段”，对内映射成受控列名，避免任意 SQL 片段进入查询
- 排序方向只保留 `ASC` / `DESC` 两种选择，不让调用方拼接表达式
- 补充稳定二级排序字段，避免同值记录跨页抖动

## 示例说明

示例中的 `Builder` 负责把接口层传入的排序字段转换成可直接用于 SQL 的 `ORDER BY` 片段。测试覆盖升序、降序以及非法字段拦截，适合作为列表查询层的基础防线。
