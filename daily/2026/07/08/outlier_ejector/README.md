# 用 outlier ejector 暂时摘除连续失败节点

同一个服务集群里，往往只有少量实例在抖动。如果客户端或网关继续把流量均匀打到这些异常节点，整体错误率会被少数坏节点放大。`outlier ejector` 的做法是对连续失败节点做短暂摘除，给它一个冷却恢复时间。

## 设计要点

- 以连续失败次数作为轻量信号，不依赖复杂监控系统也能快速落地
- 冷却窗口只做短暂摘流，避免把偶发抖动永久放大成黑名单
- 成功请求应及时清零失败计数，减少误伤健康节点

## 示例说明

示例中的 `Ejector` 用内存状态记录每个节点的连续失败次数和摘除截止时间。测试覆盖达到阈值后摘除、成功请求清零失败 streak，以及冷却期结束后自动恢复三个常见行为。

## 运行方式

```bash
go test ./daily/2026/07/08/outlier_ejector
```
