# 用时间窗和 nonce 防御 webhook 重放

支付、订阅、告警等 webhook 回调通常需要对外暴露公开入口。即使签名校验通过，如果攻击者把合法报文原样重放多次，业务也可能重复记账、重复发货或重复触发流程。因此除了验签，还需要限制时间窗并记录一次性 nonce。

## 设计要点

- 请求时间戳必须落在允许偏差窗口内，避免老报文在未来被反复利用
- nonce 只能成功消费一次，防止同一份有效报文被并发重放
- 签名串要覆盖时间戳、nonce 和请求体，避免篡改其中任一部分

## 示例说明

示例中的 `Verifier` 用 HMAC-SHA256 生成签名，并在校验时同时检查时间偏差、签名内容和 nonce 是否已使用。测试覆盖了正常请求、超时请求、重放请求和错误签名四类安全场景。

## 运行方式

```bash
go test ./daily/2026/07/08/webhook_replay_window
```
