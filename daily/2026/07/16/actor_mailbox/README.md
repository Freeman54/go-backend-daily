# 用 actor mailbox 按资源串行化任务而不阻塞全局并发

在后端服务里，很多操作只要求“同一个订单、同一个用户、同一个租户”的任务顺序执行，并不需要把所有请求都锁成单线程。`actor mailbox` 的思路是给每个 key 维护独立队列：同 key 顺序消费，不同 key 并发处理，还能用有界队列及时反馈背压。

## 设计要点

- 用 `key -> workerState` 保存每个资源的执行状态和待处理任务
- 首个任务到来时立即启动 goroutine；后续任务只入队，不重复起 worker
- 当单 key 队列达到上限时立刻返回错误，避免无限堆积

## 示例说明

示例提供 `Mailbox.Submit`，适合订单状态推进、租户级串行结算、按主键归并写入等场景。测试覆盖：

- 同一个 key 的任务执行顺序保持稳定
- 不同 key 可以并发执行
- 队列打满时返回 `ErrQueueFull`

## 运行方式

```bash
go test ./daily/2026/07/16/actor_mailbox
```
