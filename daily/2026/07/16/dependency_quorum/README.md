# 用 dependency quorum 判断服务是否还能继续接流

很多服务的依赖并不是“任意一个挂了就必须摘流”。有些依赖是硬前置，比如主数据库；有些依赖是增强能力，比如推荐、画像、审计镜像。把所有依赖都等价对待，会导致可用性过低。更合理的做法是给依赖分权重，并声明哪些是 mandatory，再按 quorum 计算是否继续接流。

## 设计要点

- mandatory 依赖失败时直接判定不可用
- 非 mandatory 依赖按权重累计，达到阈值才算通过
- 返回通过率和失败列表，便于健康检查页展示具体原因

## 示例说明

示例提供 `Evaluate`，适合 readiness、降级开关和控制面探活逻辑。测试覆盖：

- mandatory 依赖失败会直接阻断
- 权重达到阈值时服务仍可继续接流
- 权重不足时会返回失败原因

## 运行方式

```bash
go test ./daily/2026/07/16/dependency_quorum
```
