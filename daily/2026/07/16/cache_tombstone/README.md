# 用 tombstone 防止删除后的旧值重新写回缓存

缓存删除最危险的窗口不是“删不掉”，而是“删掉以后旧写请求又把脏数据写回来”。一个常见做法是在删除时留下带版本的 tombstone，后续旧版本写入必须被拒绝，直到出现更高版本的数据或者 tombstone 过期。

## 设计要点

- 删除时写入 tombstone，并记录删除版本与过期时间
- 旧版本 `Put` 遇到 tombstone 时直接拒绝，避免脏值复活
- 新版本数据可以覆盖 tombstone，兼顾一致性和恢复能力

## 示例说明

示例提供一个内存版 `Store`，适合演示“数据库先删、缓存后删”或异步回填链路里的版本保护逻辑。测试覆盖：

- 删除后的旧版本写入会被拒绝
- 更高版本写入可以正常恢复缓存
- tombstone 过期后允许重新写入

## 运行方式

```bash
go test ./daily/2026/07/16/cache_tombstone
```
