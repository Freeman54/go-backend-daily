# 用 query shape hash 给 SQL 指标降基数

数据库观测最容易失控的地方之一，是把完整 SQL 直接拿去做日志标签或 metrics label。请求里一旦带上动态数字、字符串或 `IN (...)` 参数，标签基数就会爆炸。更稳妥的方式是先把 SQL 归一化成稳定的“形状”，再计算 hash 或直接输出 shape。

## 设计要点

- 把字符串字面量、数值字面量统一替换成 `?`
- 压缩多余空白，统一大小写，避免格式差异导致 shape 漂移
- 用归一化后的文本生成短 hash，适合写入监控维度

## 示例说明

示例提供 `Normalize` 和 `Hash` 两个函数，适合在数据库中间件、SQL 采样器和慢查询上报链路中使用。测试验证：

- 仅参数值不同的 SQL 会得到相同 shape
- 查询结构变化时，shape 和 hash 都会改变

## 运行方式

```bash
go test ./daily/2026/07/16/query_shape_hash
```
