# 用 baggage allowlist 控制跨服务透传的上下文体积

链路追踪里的 `baggage` 很方便，但一旦把租户信息、实验参数、用户标签全都原样向下游透传，HTTP 头会越来越大，还可能把不该暴露的数据带到边界服务。更稳妥的方式是在入口就做 allowlist 和字节预算控制，只传真正需要的键。

## 设计要点

- 按 `key=value` 解析 W3C baggage 头，忽略非法片段
- 只保留显式允许的键，并保持原始顺序，方便排查链路
- 用最大字节数限制最终输出，避免 header 膨胀

## 示例说明

示例提供 `Filter`，适合 API gateway、服务网关或 RPC 中间件做统一清洗。测试验证：

- 未在 allowlist 中的键会被剔除
- 输出超过预算时只保留前面还能容纳的条目
- 非法片段不会污染结果

## 运行方式

```bash
go test ./daily/2026/07/16/baggage_allowlist
```
