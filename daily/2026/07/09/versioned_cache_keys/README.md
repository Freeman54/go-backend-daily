# 用版本化缓存键实现低成本批量失效

当一个分类、租户或配置域需要整体失效时，逐条删除缓存键既慢又容易漏。更常见的工程做法是把命名空间版本号编码进缓存键里，只要提升版本，就能让旧值自然失效，而无需立刻扫库删键。

## 设计要点

- 缓存键格式带上 `namespace + version + id`
- 版本号单独维护，批量失效时只做一次 bump
- 旧键可以交给 TTL 或后台清理回收，热点路径不必同步删除

## 示例说明

示例里的 `Store` 维护每个命名空间的版本号，并根据当前版本拼出缓存键。测试覆盖了当前版本读取、批量失效后旧键不可见，以及不同命名空间之间互不影响三类典型场景。

## 运行方式

```bash
go test ./daily/2026/07/09/versioned_cache_keys
```
