# 用 context.WithoutCancel 给收尾逻辑保留独立生命周期

请求超时或客户端断开后，很多收尾动作仍然应该继续执行，比如审计日志落盘、异步补偿、指标上报。如果直接复用原始请求 `context`，这些操作会被一起取消，导致尾部逻辑半途而废。

## 设计要点

- 用 `context.WithoutCancel` 继承请求内的值，但不继承取消信号
- 再额外包一层超时，给清理逻辑设置独立上限，避免后台任务泄漏
- 只把它用于必要的收尾路径，不要无差别绕过业务超时

## 示例说明

示例中的 `Detach` 保留了请求上下文里的 `request_id` 等值，同时切断了父请求的取消链路。测试覆盖了“父请求取消不影响清理逻辑”和“清理逻辑仍然有自身超时”两类关键行为。

## 运行方式

```bash
go test ./daily/2026/07/09/without_cancel_cleanup
```
