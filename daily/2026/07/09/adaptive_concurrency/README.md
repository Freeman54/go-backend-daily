# 用 adaptive concurrency 按延迟反馈动态收缩并发

固定并发上限在流量和依赖状态变化时往往过于僵硬。更稳妥的做法是根据最近请求的延迟和失败信号动态调节允许并发，在系统变慢时主动降载，在系统恢复时逐步放量。

## 设计要点

- 延迟超过目标阈值或请求失败时立刻减半并发，优先止损
- 延迟稳定且明显低于目标时再逐步加一，避免过冲
- 最小值和最大值兜住边界，防止抖动把系统打到 0 或无限膨胀

## 示例说明

示例里的 `Limiter` 使用类似 AIMD 的策略：坏信号快速回退，好信号缓慢增长。测试覆盖了稳定低延迟放量、高延迟收缩，以及失败信号触发降载三类后端常见场景。
