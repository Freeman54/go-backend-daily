# 用事务死锁重试兜住数据库瞬时冲突

高并发写入下，数据库偶发死锁或锁等待超时并不罕见。对于这类瞬时冲突，直接把错误抛给上层会造成大量本可成功的失败请求，更稳妥的做法是在事务边界内识别可重试错误并有限退避重试。

## 设计要点

- 只重试明确可恢复的冲突，例如 `40001`、`40P01`、deadlock、lock wait timeout
- 限制最大尝试次数和退避上限，避免把数据库压得更慢
- 把重试封装在事务执行器里，让业务函数只关心一次事务逻辑

## 示例说明

示例里的 `Policy` 用 SQLState 和错误文案双重识别瞬时锁冲突，并以指数退避执行有限次重试。测试覆盖了“死锁后重试成功”“非重试错误立即返回”和“调用方上下文已取消时快速退出”。

## 运行方式

```bash
go test ./daily/2026/07/09/tx_deadlock_retry
```
