# 用 visibility heartbeat 保护长耗时消息消费

消息队列的可见性超时如果短于任务执行时间，同一条消息会在消费者尚未处理完成时被重新投递，造成重复执行。稳妥的做法是在处理期间定期续租，但又必须给续租设置上限，避免异常任务无限占着消息不释放。

## 设计要点

- 只在接近过期时续租，减少不必要的续约调用
- 每次按固定可见性窗口延长，简化消费者实现
- 设置最大续租时长，超过上限就放弃续租并交给重试或死信流程

## 示例说明

示例里的 `Lease` 模拟了消息消费者的可见性租约：`Due` 判断是否进入续租窗口，`Heartbeat` 在接近过期时延长租期，并在超过最大续租上限时返回放弃信号。测试覆盖了续租窗口判断、成功续租和达到上限后停止续租。

## 运行方式

```bash
go test ./daily/2026/07/09/visibility_heartbeat
```
