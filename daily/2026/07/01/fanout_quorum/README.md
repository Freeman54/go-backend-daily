# 用 quorum 读取降低副本抖动影响

读多写少的接口常常会同时访问多个副本。单个副本抖动时，如果串行等待，很容易把尾延迟放大。这个示例并发访问多个副本，只要拿到满足 `quorum` 的成功响应就立即返回。

## 设计要点

- 用子 `context` 在达到法定成功数后取消慢副本
- 当剩余副本数量已不足以凑齐 `quorum` 时尽早失败
- 通过 `quorum` 平衡可用性和一致性要求

## 示例说明

业务里可以把 `Replica` 对接成缓存节点、只读库或搜索分片查询函数。对延迟敏感的查询，优先追求“足够多的成功结果”，通常比等待全部副本更稳。

## 运行方式

```bash
go test ./daily/2026/07/01/fanout_quorum
```
