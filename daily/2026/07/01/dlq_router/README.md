# 用重试与死信路由保护消息消费

消息消费失败时，如果一律无限重试，会把消费组卡死；如果一律丢弃，又会吞掉可恢复故障。这个示例把错误分成可重试的瞬时故障和不可恢复的脏消息，再根据尝试次数决定重试还是送入死信队列。

## 设计要点

- `ErrTransient` 代表网络抖动、下游超时等可恢复故障
- `ErrPoison` 代表数据格式错误、关键字段缺失等不可恢复故障
- 达到最大尝试次数后把瞬时故障也转入 DLQ，避免消息永久阻塞

## 示例说明

把这个分类器接到 Kafka、RocketMQ 或 RabbitMQ 的消费逻辑时，可以统一决定 `ack`、重试和死信策略，降低业务代码里的分支复杂度。

## 运行方式

```bash
go test ./daily/2026/07/01/dlq_router
```
