# 读副本做会话黏性，减少跨副本抖动带来的缓存未命中

很多读多写少的系统会把查询流量分发到多个副本，但如果同一个会话在短时间内不断切副本，容易出现页缓存未热、连接池冷热不均、读后写感知波动等问题。一个折中做法是在健康前提下给会话短时间“黏”住同一副本。

## 设计要点

- `Router` 为每个 session 记录一个短 TTL 的副本绑定
- 只有副本健康且复制延迟不超过 `MaxLag` 时，绑定才会继续沿用
- 当前副本变慢或超 lag 后，会自动切到下一台健康副本

## 示例说明

示例先把 session 绑定到 `replica-a`，即使下一次传入的副本顺序变化，只要 `replica-a` 仍健康，就继续命中同一台；当它的延迟超阈值后，路由会自动迁移到 `replica-b`。

## 运行方式

```bash
go test ./daily/2026/07/14/replica_stickiness
```
