# 用 slow start limiter 控制恢复后的并发爬升

下游服务刚从故障中恢复时，如果上游立刻把满负载流量全部打回去，常见结果是刚恢复又被压垮。和 TCP slow start 类似，服务端也可以先从很小的并发开始，确认连续成功后再逐步放量。

## 设计要点

- `Limiter` 初始只放行 1 个并发窗口，避免恢复瞬间被尖峰打穿
- 每累计 `step` 次成功才把限额加 1，形成平滑爬升
- 一旦出现失败就立刻回退到 1，重新进入保守恢复模式

## 示例说明

这个示例适合放在任务调度器、批处理 worker 或下游连接池前面，作为恢复阶段的附加保护。测试覆盖了稳定成功后的逐步放量，以及失败后的快速回退。

## 运行方式

```bash
go test ./daily/2026/07/14/slow_start_limiter
```
